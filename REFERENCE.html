Thoroughbred Basic Interpreter
Reference Manual

This manual provides a detailed alphabetical overview of all implemented directives (commands) and functions in this interpreter.

Configuration and Disks

The interpreter uses a 'Disk' system for file management, allowing logical disk names (like D0, D1) to map to physical directories on the host system.

IPLINPUT

On startup, the interpreter reads the configuration file IPLINPUT from the current working directory.

Format: 
DISKNAME = /path/to/directory
Example: 
D0 = ./data/primary
D1 = ./data/backup

For file operations (like OPEN), the system searches in the order: D0, D1, etc., unless a specific path or disk number is provided.

Creating Files

disk number to specify where the file should be created.

File Formats

The interpreter supports four specific file types:

DIRECT (Keyed File)

Fixed-record-length files accessed via a unique string key.

  • Optimized for Random Access.
  • Create: DIRECT "name", key_len, rec_len, disk

SORT (Sort File)

Contains only keys (up to 128 bytes). Used for sorting or indexing.

  • Create: SORT "name", key_len, folder_len, disk

INDEXED

Key-Value storage mechanism.

  • Create: INDEXED "name", 10, 0, disk

SERIAL

Sequential files. Records are written chronologically.

  • Use for logs or sequential data.
  • Create: SERIAL "name", rec_len, disk

A

ABS

Numeric Function
ABS(num)

Returns the absolute value of the number num.

Examples: ABS(-5) → 5

ASC

String Function
ASC(string)

Returns the decimal ASCII value of the first character of string.

Examples: ASC("A") → 65

ATH

Conversion Function
ATH(hex_string)

Converts a hexadecimal string to a binary string (ASCII). If the input string has an odd length, a '0' is prepended automatically.

Examples: ATH("41") → "A"

ATN

Numeric Function
ATN(num)

Returns the arctangent of num in radians.

C

CALL

Flow Control Directive
CALL "program", [ERR=line], [args...]

Starts the execution of an external BASIC program as a subroutine.

  • Creates a new execution context. Variables from the caller are not visible to the called program unless explicitly passed.
  • The called program uses ENTER to receive arguments.
  • EXIT returns control to the caller (instruction after CALL).

CHR$

String Function
CHR$(code)

Returns the character corresponding to the ASCII code (0-255).

CLOSE

I/O Directive
CLOSE (channel)

Closes the file opened on channel. Flushes all buffers to disk.

COS

Numeric Function
COS(num)

Returns the cosine of angle num (in radians).

CVS

String Function
CVS(string, mode)

Converts/manipulates string based on bitwise mode:

  • 1: Strip leading spaces (LTRIM)
  • 2: Strip trailing spaces (RTRIM)
  • 16: Convert to Uppercase
  • 32: Convert to Lowercase

D

DIM

Declaration Directive
DIM var[size](len), ...

Reserves memory for arrays. Format (len) is optional for strings (dynamic).

DIRECT

File Directive
DIRECT "filename", key_len, rec_len, disk_num

Creates a new 'DIRECT' (Keyed) file on the specified disk.

DTN

Date Function
DTN(string [, mask])

Converts a date string to a Julian Day Number via `mask`.

E

END

Flow Control Directive
END

Terminates program execution immediately and closes files.

ENDTRACE

Debug Directive
ENDTRACE

Disables trace mode.

ENTER

Flow Control Directive
ENTER var1, var2...

Receives arguments in a program started by CALL.

ERASE

File Directive
ERASE "filename"

Physically deletes a file from disk.

EXECUTE

Meta Directive
EXECUTE string_expr [, OPT="LOCAL"]

Compiles and executes BASIC code contained in string_expr at runtime.

EXIT

Flow Control Directive
EXIT

Exits a CALLed program. Writes back modified variables (via ENTER) to the caller.

EXP

Numeric Function
EXP(num)

Returns e raised to the power of num.

EXTRACT / EXTRACTRECORD

I/O Directive
EXTRACT (chn, KEY=k, IND=i) vars...

Reads record data and locks the record.

  • Important: Does not advance the file pointer. A subsequent WRITE updates the same record.

F

FIND

I/O Directive
FIND (chn [, KEY=k, IND=i]) [vars...]

Checks for a record and optionally reads data.

  • Behaves like READ, except that if the key is not found, the file pointer is not moved.
  • If the record is found, the file pointer is updated to that record.
  • On SORT files, it verifies the existence of the key but does not transfer data.

FOR

Flow Control Directive
FOR var = start TO end [STEP step]

Starts a loop counter. Ends with NEXT var.

FPT

Numeric Function
FPT(num)

Returns the fractional part of num.

G

GOSUB

Flow Control Directive
GOSUB line

Jumps to subroutine at line. Returns with RETURN.

GOTO

Flow Control Directive
GOTO line

Unconditional jump.

H

HTA

Conversion Function
HTA(string)

(Hex to ASCII) Converts bytes in string to hex representation.

I

IF

Flow Control Directive
IF condition THEN line / statement

Executes action if condition is true. Operators: =, <>, <, >, <=, >=.

INDEXED

File Directive
INDEXED "filename", ...

Creates an INDEXED file (Key-Value store behavior).

INPUT

I/O Directive
INPUT [chn, opts] "Prompt: ", vars...

Requests user input or reads from terminal. Supports IOL= formatting.

INT

Numeric Function
INT(num)

Returns the floor of num (largest integer <= num).

IOLIST

I/O Directive
line IOLIST item [, item ...]

Defines a reusable template for I/O operations.

Items:

  • Variables: A$, B (Read/Write values)
  • Mnemonics: 'CS', 'LF' (Terminal only)
  • Cursor: @(col, row) (Terminal only)
  • Literals: "Text" (Output/Expect text)
  • Skip: * (Skip field on input)

Usage: READ (1, IOL=100)

IPT

Numeric Function
IPT(num)

Returns the integer part (truncation) of num.

K

KEY

I/O Function
KEY(channel [, ERR=line])

Returns the key of the next record without reading it.

L

LCS

String Function
LCS(string)

Converts string to Lowercase.

LEN

String Function
LEN(string)

Returns string length.

LET

Assignment Directive
LET var = expr

Assigns value. Keyword LET is optional.

LOG

Numeric Function
LOG(num)

Natural logarithm.

M

MAX / MIN

Function
MAX(items...) / MIN(items...)

Returns largest/smallest value.

MOD

Numeric Function
MOD(num, div)

Returns remainder of division.

N

NEXT

Flow Control Directive
NEXT var

Ends a FOR loop.

NOT

String Function
NOT(string)

Bitwise NOT on string bytes.

NUM

Conversion Function
NUM(str [, NTP, SIZ...])

Converts string to number with validation options.

O

ON ... GOTO / GOSUB

Flow Control Directive
ON val GOTO line1, line2...

Conditional jump based on integer index `val`.

OPEN

I/O Directive
OPEN (chn) "file" [, options]

Opens a file on channel `chn` (1-255). Channel 0 is terminal.

OR

String Function
OR(str1, str2)

Bitwise OR.

P

POS

String Function
POS(haystack, needle) / POS(relation)

Finds position of substring or condition match.

PRINT

I/O Directive
PRINT (chn) expr...

Outputs to channel. Support Mnemonics on terminal.

R

READ

I/O Directive
READ (chn [, KEY=k, IND=i]) vars...

Reads fields from a record.

  • Sequential: Reads next record, advances pointer.
  • Keyed: Reads specific record.
  • DOM=line: Jump on Duplicate/Missing Key.
  • END=line: Jump on EOF.

REM

Comment
REM text

Comment line. Ignored.

REMOVE

File Directive
REMOVE (chn [, KEY=k])

Deletes a record. If no KEY, deletes last accessed record.

RETRY

Flow Control Directive
RETRY

Retries the statement that caused the last error.

  • Returns execution to the line where the error occurred.
  • Restores the SETERR setting to its state before the error.

RETURN

Flow Control Directive
RETURN

Returns from GOSUB.

RND

Numeric Function
RND([seed])

Returns random float 0.0-1.0.

ROUND

Numeric Function
ROUND(num, precision)

Rounds a number.

S

SELECT

I/O Directive
SELECT (chn) ...

Partial implementation of record filtering.

SERIAL

File Directive
SERIAL "name", ...

Creates a SERIAL (append-only/seq) file.

SETERR

Flow Control Directive
SETERR line | OFF | ON

Defines a global error-handler routine.

  • Transfers control to line if a runtime error occurs (and is not handled locally by ERR=).
  • SETERR OFF disables the handler.
  • Error handler is automatically disabled upon entry.

SETTRACE

Debug Directive
SETTRACE [(chn)]

Enables execution logging.

SET TRACEMODE

Debug Directive
SET TRACEMODE "mode"

Configures trace (FULL, PARTIAL, SKIPCALLS).

SGN

Numeric Function
SGN(num)

Returns sign (-1, 0, 1).

SIN/COS/TAN

Numeric Function
SIN(n)

Trig functions.

SORT

File Directive
SORT "name"...

Creates a SORT file.

SQR

Numeric Function
SQR(num)

Square root.

STOP

Flow Control Directive
STOP

Halts program.

STR$

Conversion Function
STR$(num)

Number to string conversion.

U

UCS

String Function
UCS(string)

Converts to Uppercase.

V

VAL

Conversion Function
VAL(string)

String to number conversion.

W

WRITE

I/O Directive
WRITE (chn [, KEY=k]) vals...

Writes data to a record (Update or Insert).

X

XOR

String Function
XOR(str1, str2)

Bitwise XOR.

Terminal Mnemonics

Supported mnemonics for PRINT statements:

Mnemonic Function Effect
'CS' Clear Screen Clears screen, moves cursor to top-left.
'BR' Begin Reverse Starts reverse video.
'ER' End Reverse Stops reverse video.
'BU' Begin Underline Starts underline.
'EU' End Underline Stops underline.
'VT' Vertical Tab Cursor Up.
'LF' Line Feed Cursor Down.
'BS' Backspace Cursor Left.
'CH' Cursor Home Moves cursor to top-left (no clear).
'CE' Clear to EOS Clears to end of screen.
'CL' Clear to EOL Clears to end of line.
'LD' Line Delete Deletes current line.
'@'(c,r) Cursor Pos Moves cursor to column c, row r.
